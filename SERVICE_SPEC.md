# henro-log（遍路ログ） - サービス仕様書

## 1. サービス概要
- **一言で**: 四国八十八ヶ所を「少しずつ」巡る歩みを、訪問1件ずつのイベントとして記録し、同行者や前回からの間隔まで横断的に振り返れる巡拝記録アプリ。
- **対象ユーザー**: 区切り打ち（複数回に分けて巡る）でお遍路を回る人。まずは作者自身（個人用MVP）。
- **解決する課題**: 既存アプリは「札所に日付・写真を貼る」モデルで、"誰と行ったか""前回から何年ぶりか"といった文脈・関連性を構造化して振り返れない。henro-log は訪問をイベントとして記録し、同行者軸・再訪間隔で巡拝の物語を残す。
- **ステータス**: MVP

## 2. ユーザーロールと権限
| ロール | できること |
|---|---|
| 利用者（本人・要ログイン） | Google またはメール/パスワードでログイン後、訪問記録の作成・編集・閲覧、集計の閲覧。参拝データは現状は端末内ローカル保持（クラウド同期は将来対応）。 |

## 3. 機能一覧
### 実装済み
- [x] 札所（縦スクロールの巡拝トレイル／押印） — アプリのホーム（`/`・「札所」タブ）＝88札所を**1つずつ縦に連ねた巡拝トレイル**（点線の巡拝路で連結、道場別に4区切り、長い分はスクロール）。各行に **番号＋寺名＋所在市・ご本尊＋最終参拝日/回数**（未参拝は「未参拝」）を表示。**行をタップすると確認シートが開き（未参拝＝「この札所に参拝を記録しますか？」＋記録する／キャンセル。誤タップ防止のため即記録はしない／参拝済＝今日また参拝・詳細写真へ・最新を取り消し）、記録するを押すと今日の日付で参拝を1件記録**し御朱印風「済」スタンプが押される（記録時にファミコン風の押印効果音）。確認シートには札所ごとの挿絵（お堂・三重塔・石灯籠・本堂の4種を番号で割当）を表示。**シートで「📷 写真を追加（任意）」から撮影/選択した写真を押印と同時に記録**でき（保存前に自動縮小）、記録直後に記念フレーム付きで全画面表示。写真はその参拝の photoIds に紐づき写真タブ/札所詳細に載る。記録せずシートを閉じた場合は選択中の写真を自動破棄。複数回参拝は回数バッジ。上部の進捗は「○/88」の道の上を**お遍路さん（住職）が歩いて進むアニメーション**（2コマ＋上下バウンド、進捗％の位置まで前進）＋残数メッセージ＋％。**番号・「済」は御朱印/納経帳ふうの毛筆体**（Yuji Syuku をサブセット同梱＝約5KB・オフライン対応）で表示。**巡礼を楽しくする演出**＝巡拝路は一直線ではなく左右に蛇行しながら下る（サイン波・道場ごとに連続する滑らかな曲線、SVG）／背後に四国八十八か所の絵地図を薄く（opacity 0.12）敷く／スクロール中はヘッダー・フッターを透明にして背景の絵地図を見せる（停止で白に戻る）／歩いた分（現在地まで）が朱色に色づく／次に行く札所に「つぎ」パルス／道場ごとに進捗（○/N）と満願バッジ／節目（初参拝・道場満願・八十八結願）で花吹雪＋特別トースト／押印は手押しっぽい微回転
- [x] 訪問記録の作成/編集/削除 — 盤タップで今日の参拝を即記録。写真・心境メモ・納経有無つきの詳しい記録/編集は札所詳細→フォーム（`/visit/new?temple=`, `/visit/:id`）で行う（同行者/天候/移動手段は廃止）
- [x] 写真（カメラ） — 撮影/選択した写真を端末内（IndexedDB Blob）に保存。フォームでサムネイル/削除、札所詳細で表示
- [x] 写真の自動縮小 — 保存前に長辺1600px以内へ縮小しJPEG(品質0.82)で再エンコード。EXIF回転を吸収。再エンコードで増える場合は原本を保持（`src/lib/downscaleImage.ts`）。実測で3000×2000写真が約77%削減
- [x] 写真ビューア（記念フレーム） — サムネタップで全画面表示。「第○番 寺名・参拝日・朱印風"巡拝"」を添えた記念フレーム。スワイプ/矢印/Escで操作
- [x] 写真アルバム — 全札所の写真を時系列グリッドで一覧（/album タブ）。タップで全画面ビューアへ
- [x] 記念フレーム画像のシェア/保存 — ビューアから写真＋和紙フレーム（寺名・日付・朱印風）を1080x1350 PNGに合成し、Web Share APIで共有（非対応時DL）。SNS拡散による集客導線＝競合優位の核
- [x] タイムライン — 全訪問を時系列で表示
- [x] 再訪間隔の算出 — 同一札所の訪問履歴から「前回から○年ぶり・○回目」を自動表示
- [x] PWA化 — インストール可能・オフライン（vite-plugin-pwa）。新バージョン検知時はアプリ内に「新しいバージョンがあります → 更新」通知（`registerType: 'prompt'`, `src/components/PwaUpdatePrompt.tsx`）を表示し、タップで最新版へ切替（キャッシュ更新の取りこぼしを防止）
- [x] ユーザー認証（ログイン必須ゲート） — Firebase Authentication による Google ログイン ＋ メール/パスワード（新規登録／ログイン）。未ログイン時はログイン画面を表示しアプリ本体をガード。設定「アカウント」にログイン中のアカウント（写真/名前/メール）とログアウトを表示（`src/auth/*`, `src/pages/Login.tsx`, `src/lib/firebase.ts`）
- [x] 現在地から近い札所（地図タブ） — 端末のGPS（`navigator.geolocation`）で現在地を取得し、全88札所との直線距離（ハバーサイン）を計算して**一番近い札所＋近い順リスト（上位5件）**を表示。各札所に**「Googleマップで経路案内」**（`https://www.google.com/maps/dir/?api=1&destination=緯度,経度&travelmode=walking` を新規タブで開く＝出発地は現在地・徒歩モード。API キー/請求登録は不要）と札所詳細への導線、参拝済み/未参拝バッジ。位置情報は端末内でのみ使用し送信しない。許可拒否/取得失敗/非HTTPS時は日本語メッセージ＋再試行（`src/pages/NearbyView.tsx`, `src/lib/geo.ts`）
- [x] プロフィール編集 — 表示名・連絡先を端末内に保存（未入力ならログイン中アカウントの名前・メールで初期化）
- [x] 音楽（BGM）ON/OFF — 設定でファミコン風の巡礼テーマBGMを ON/OFF。ON にすると全画面でループ再生（設定は localStorage に保存、自動再生ブロック時は次のタップで再生開始）
- [x] 満願お祝いビュー — 八十八ヶ所すべて達成した瞬間に、全画面で花火＋新規ファンファーレ＋「満願成就」カードを表示（どの画面からでも発火。起動時に既に満願でも自動表示はしない）
- [x] 一括押印／リセット（デモ用） — 設定の「デモ・お試し」から、未参拝の全札所に今日の日付で一気に押印（→満願お祝いが流れる）／全参拝記録の消去ができる

## 5. データモデル
### コレクション / テーブル一覧（IndexedDB / Dexie）
- `temples` — 札所マスタ（88件固定・シードで内蔵）
  - 主要フィールド: id(番号1-88), name(寺名), sect(宗派), honzon(ご本尊), prefecture, city, lat, lng
  - lat/lng は全88件投入済み（Wikipedia座標, WGS84）。15/17/18/19番は誤り修正済み
- `visits` — 訪問記録（記録の中心。1札所につき何件でも）
  - 主要フィールド: id, templeId, visitedOn(訪問日), photoIds[], note(心境メモ), nokyo(納経有無 bool), createdAt, updatedAt
  - 廃止フィールド: companionIds[]（常に空配列で保存）, weather, transport（スキーマは残置）
- `photos` — 写真（Blob 保管）
  - 主要フィールド: id, blob, mime, createdAt
- `profile` — 利用者プロフィール（単一レコード id=1。Dexie version 2 で追加）
  - 主要フィールド: id(=1), name, email
- `companions` — 同行者マスタ（機能廃止。スキーマのみ残置）

### 派生（計算で導出）
- 札所の訪問回数 / 最終訪問日 … `visits` を templeId で集約
- 前回からの間隔・○回目 … 同一 templeId の `visits` を visitedOn 順に並べて差分

## 6. 外部連携
- 認証: **Firebase Authentication**（Google ログイン ＋ メール/パスワード）。ログイン必須（未ログインはログイン画面をゲート表示）。Firebase プロジェクト = `henro-log-ath`
- DB: IndexedDB（Dexie.js）。参拝データは引き続き端末内ローカル保持（クラウド同期は将来対応）
- 位置情報: ブラウザ標準 `navigator.geolocation`（端末内で完結・座標は外部送信しない）
- 地図: **Google マップへのリンク方式**（`https://www.google.com/maps/...` を新規タブで開く）。Maps JavaScript API のキー・請求先アカウントは不要
- API: なし（札所マスタは静的JSON同梱）
- ホスティング: 静的ホスティング（GitHub Pages: https://tsubasagit.github.io/henro-log/ ・PWAとして配信）

## 7. ビジネスルール
- 1つの札所に対し訪問記録（visit）は複数作成できる（区切り打ち・お礼参り対応）
- 訪問日（visitedOn）は必須。写真・メモ・納経は任意
- 「前回からの間隔」は同一札所の直近2件の visitedOn 差から算出

## 8. 非機能要件
- 想定ユーザー数: 1（MVP）。将来的に個人ごとローカル完結の複数ユーザー
- パフォーマンス目標: 88札所一覧・数百件の訪問記録を即時表示
- セキュリティ: ログイン必須（Firebase Auth）。参拝データ本体は現状は端末内に保持しクラウド送信なし（クラウド同期の実装時に Firestore セキュリティルールで uid 単位のアクセス制御を行う）
- オフライン: PWAでオフライン記録・閲覧が可能であること

## 9. 既知の課題・制限
- MVPはローカル完結のため、機種変更時はエクスポート/インポートが必要
- 複数端末間の同期は未対応（将来のクラウド同期で検討）
- 別格二十霊場・他霊場は対象外（四国八十八ヶ所のみ）
